## 附加确认项（本次开发中发现，《00》未覆盖，报请知悉/拍板）

A. **历史夏令时**：IANA 时区库如实包含中国 1986–1991 夏令时（UTC+9）。引擎按真实钟表时换算（1990 年夏 14:00 → 真太阳时 12:45 午时），比忽略夏令时的市面软件更准，但与用户拿其他软件对照时会出现"不一致"观感。建议：保持现状（严谨第一），由模型B在"溯源"层展示修正明细（真太阳时确认页本就要展示）。**如需提供"忽略夏令时"兼容开关，请拍板。**

B. **1940 年代命例**：民国/沦陷区时制在 IANA 记录不全，该时期出生时间的"钟表时→UTC"换算存在 ±1 小时输入侧风险。M1 范围内影响极小（目标用户出生年代），报备知悉即可。

C. **时辰反推问题权重表 v1.0**：12 时辰似然权重为通行常识近似（天色/饭点/作息），已文档化（《时辰反推置信度算法.md》），**建议与拍板项1、2 一并交命理顾问校准签字**。

D. **契约响应示例的日柱值**：《00》4.1 示例 JSON 中 1990-06-15 标注日柱"丙子"，实际为"辛亥"（双源一致）。判断为示例笔误，schema 结构不受影响，已按 schema 逐字段实现；报备免生疑义。
