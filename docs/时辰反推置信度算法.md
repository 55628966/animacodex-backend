## 一、问题模型与贝叶斯更新

### 1.1 假设空间与先验

假设空间为 12 个时辰地支（固定顺序）：

> 子(23-01) 丑(01-03) 寅(03-05) 卯(05-07) 辰(07-09) 巳(09-11) 午(11-13) 未(13-15) 申(15-17) 酉(17-19) 戌(19-21) 亥(21-23)

无任何证据时采用**均匀先验**：

```
P₀(b) = 1/12 ≈ 0.0833，b ∈ 12地支
```

### 1.2 逐答案乘法贝叶斯更新

每个问题的每个选项对应一个 12 维**相对似然向量** `L[q][o]`，表示
`P(用户选该选项 | 出生于该时辰)` 的相对大小。用户每答一题，做一次乘法更新并归一化：

```
P_new(b) ∝ P_old(b) × L[q][o][b]
P_new(b) = P_old(b) × L[q][o][b] / Σ_b' ( P_old(b') × L[q][o][b'] )
```

- 对外输出的 `confidence` 四舍五入到 **4 位小数**；锁定判断使用**未舍入值**（避免 0.89995 被舍成 0.9000 后误锁）。
- 似然表**不含 0**，因此任何回答都不会把某时辰置信度打成绝对零，也不会出现除零。
- 乘法可交换 → 同一组有效答案，更新顺序不影响最终分布。

### 1.3 重复回答与非法输入

- **同一问题重复回答：以最后一次为准。** 实现方式：全量重算时按序覆盖（`question_id → option_key` 后写覆盖先写），再统一做乘法更新。`asked_count` 按**去重后的问题个数**计。
- 非法 `question_id` → `ValueError("UNKNOWN_QUESTION")`；非法 `option_key` → `ValueError("UNKNOWN_OPTION")`。缺字段按同样规则处理。

## 二、锁定 / 问尽判定（红线，模块内执行）

设去重后已答问题数为 `asked_count`，未舍入最高后验为 `max_conf`，判定顺序如下：

| 条件 | locked | next_question_id | 说明 |
|---|---|---|---|
| `max_conf ≥ 0.90` | `True` | `None` | 锁定时辰，允许全四柱排盘（可在第5问之前提前锁定） |
| `asked_count ≥ 5` 且未达 0.90 | `False` | `None` | **红线：问尽。永不锁定，上层只允许前三柱排盘（partial=true）** |
| 其余情况 | `False` | 判别力最大的未问问题ID | 继续提问 |

《00》4.2 v1.1 增补（拍板 2026-07-17）：`locked=True` 时状态中**必须**同时返回
`"locked_branch": "<地支>"`（未舍入后验最大项对应的时辰，纯确定性）；未锁定状态不得出现该字段。
API `answer` 响应原样透传。

**这条判断在本模块（后端）内做，前端不许绕过。** 后端拿到 `locked=False` 且
`next_question_id=None` 时，必须按契约返回 `"locked": false`，模型B必须展示
"无法锁定时辰"告知文案并只出前三柱解读。

## 三、下一问选择策略：期望信息增益（EIG）

采用**期望信息增益**（两种可选方案中选了这种，非代理指标）：

1. 回答生成模型：把该问题各选项的似然按时辰**列归一化**，作为该时辰下用户选各选项的条件概率：
   `P(o|b) = L[q][o][b] / Σ_o' L[q][o'][b]`
2. 选项边际概率：`P(o) = Σ_b P(b) × P(o|b)`（`P(b)` 为当前后验）
3. 期望信息增益（香农熵，自然对数）：

```
EIG(q) = H(当前后验) − Σ_o P(o) × H(回答 o 后的后验)
```

在**未问过**的问题中取 EIG 最大者；并列时取问题库中**靠前**者（固定遍历顺序，保证确定性）。

说明：EIG 计算中使用列归一化的 `P(o|b)` 作为回答模型；实际置信度更新则直接乘相对似然 `L`（两者在列和不完全相等时有细微差异，属 v1.0 记录在案的近似——EIG 只用于**选题排序**，不影响置信度数值本身）。"不确定"占比高的问题增益自然偏低，会被排到后面。

## 五、可复现性声明

- 模块为**纯函数**：无网络、无IO、**禁用随机数**；`update_session(answers)` 每次从均匀先验全量重算。
- **同一 answers 序列 → 永远同一结果**（含 candidates 逐位、next_question_id、locked）。回归测试 `tests/test_hour_inference.py::test_determinism_same_answers_same_result` 守护该性质。
- EIG 选题遍历顺序固定（问题库定义序），并列取靠前者，无任何不确定来源。

## 六、局限性（v1.0 近似，记录在案）

1. **天色/日出日落受季节与纬度影响**：卯时冬天可能仍黑、夏天已亮；v1.0 用全年平均口径近似，未按出生月份调节（后续版本可按节气微调向量）。
2. **饭点、劳作、午睡等社会作息随年代与城乡差异漂移**：v1.0 按传统农耕社会口径取值，对城市/近年出生者判别力偏弱。
3. **相邻时辰天然难分**（如午/未、子/丑）：保守量级下常需多条独立线索方能锁定，5 问内锁不住属预期行为，红线兜底。
4. **EIG 的回答模型用列归一化似然**，与置信度更新的原始似然存在细微口径差（见第三节说明），只影响选题顺序不影响置信度。
5. **全部权重为 v1.0 判别力权重，已拍板 2026-07-17，命理顾问会签待补**；会签前任何对外解读须注明"待校准"。

## 七、给模型B的接口清单（问题ID + 语义）

模型B据此写用户文案；**问题ID与语义不许改**，文案怎么写归B，但语义指向必须与下表一致。选项语义见第四节各表"语义"列（option_key 同样不许改）。

| question_id | 语义（evidence） | 主要判别方向 |
|---|---|---|
| `q_sky_light` | 出生那一刻屋外天色的明暗程度（大亮/全黑/蒙蒙亮或擦黑） | 昼 vs 夜（全库区分度最强） |
| `q_family_asleep` | 出生时家中大人的作息状态（酣睡/都醒着/午间打盹） | 深夜 vs 白昼；午睡指向"午" |
| `q_meal` | 出生时间与三餐饭点的关系（早饭/午饭/晚饭前后，或不在饭点） | 辰、午、酉三个饭点锚定 |
| `q_rooster_sun` | 出生时是否临近鸡鸣头遍、日出或日落等自然标志 | 丑寅（鸡鸣）、卯（日出）、酉（日落） |
| `q_work` | 出生时家中大人的劳作状态（干活中/刚收工/歇晌/深夜俱寂） | 白昼劳作段 vs 收工(酉戌) vs 深夜 |
| `q_sun_position` | 出生时太阳在天上的大致位置（正头顶/偏东低/偏西低/看不见） | 白昼内部：上午/中午/下午 |
| `q_night_watch` | 若出生在夜里，大约处于哪个更次（一更/三更/五更） | 夜间内部：戌 / 子丑 / 寅 |

交互约定（复述契约，防绕过）：B 每收到一个答案就把**完整答案列表**传给后端 `answer` 接口；`locked=false` 且 `next_question_id=null` 时，B 必须展示"无法锁定时辰"的告知文案并只出前三柱解读，**不许在前端自行放宽**。
