## 1. 节气源

- **主源：Swiss Ephemeris（pyswisseph，Moshier 太阳模型）**。节气定义为太阳视黄经到达 15° 整数倍的精确交越时刻，直接以 UT 计算，时刻精度秒级（远超契约"分钟级"要求），覆盖 1800–2399，无网络、无数据文件依赖，任何机器可复现。
- 交叉验证副源：lunar-python 1.4.8（独立实现）。全范围（1901–2099 全部"节"）实测两源偏差 ≤ 94 秒（集中于远期年份的 ΔT 模型差异，1900–2050 内 ≤ 55 秒）。
- **弃用 sxtwl 作主源的原因（重要）**：实测发现 sxtwl 输出的是"当时中国钟表时"，1940 年代含历史时制（如 1944 年小寒 19:41，比天文真值 18:39（UTC+8）晚整 1 小时）。已用瑞士星历表独立仲裁确认（1944-01-06 小寒 = 10:39:15.6 UT）。sxtwl 降级为第三对照源。
- `meta.solar_terms_source` 逐结果写入上述主源描述。

## 2. 边界口径

- **年柱以立春精确时刻为界**（非农历正月初一）；**月柱以十二"节"精确时刻为界**（非中气）。
- 比较方式：出生绝对时刻（钟表时间 + IANA 时区 → UTC）与节气交越绝对时刻（UT）直接比较，**与出生地经度无关**（节气是全球同一瞬间的天文事件）。
- 引擎内部测试保证：节交接前后 ±1 秒，年柱/月柱必然正确切换（tests/test_cross_validation.py::test_engine_internal_term_boundary）。

## 3. 真太阳时

- 公式：真太阳时 = 当地钟表时间 + 经度修正 + 均时差；经度修正 = 经度×4分钟 − 时区偏移；均时差用 NOAA/Spencer 级数（精度约 ±30 秒）。
- 契约字段：`correction_minutes` = 总修正（含均时差），`equation_of_time_minutes` = 均时差分量（与《00》示例算式一致：钟表时 + correction = 真太阳时）。
- **历史时制如实处理**：IANA 时区库含中国 1986–1991 夏令时（UTC+9）。例：1990-06-15 14:00 北京钟表时，真太阳时为 12:45（午时）——忽略夏令时的排盘软件会错判未时。这是本引擎的严谨性特性。
  - 已知限制：1940 年代沦陷区/民国时制在 IANA 中记录不全，该时期命例存在 ±1 小时的输入侧风险（见拍板项报告附加项）。

## 4. 日柱与时柱

- **以真太阳时定时辰**（契约要求）；日界为真太阳时 00:00。
- **晚子时（真太阳时 23:00–24:00）**【已拍板 2026-07-17，命理顾问会签待补·拍板项1】：配置开关 `late_zi_new_day`——
  - `true`（换日派）：日柱取次日；
  - `false`（不换日派）：日柱留当日；
  - 两派下晚子时的**时柱天干均按次日日干五鼠遁**（与 lunar-python 已发布流派语义一致，两派时柱相同）。
  - 默认值 `true`（换日派）为**正式默认**（Owner 拍板 2026-07-17，依据主流做法；不换日开关保留，不对用户暴露）。
- 日柱基准锚点：1990-06-15 = 辛亥日（双源核验），由此按日序推算，全范围与 lunar-python 交叉验证一致。

## 7. 大运

- 顺逆：阳年男/阴年女顺排，阴年男/阳年女逆排（年干阴阳以立春界精确判定）。
- 起运：顺排取出生→下一节、逆排取上一节→出生的精确时间差，**取整到分钟**后按 4320 分钟（3天）=1岁、360 分钟（1天）=4个月→1月、12 分钟（1时辰÷10）=1天 逐级换算（与 lunar-python "流派2/分钟法"口径一致）。
- 输出：`start_age`（几岁几个月），`start_date`（出生日 + 岁/月/天 的公历日历加法）；8 步大运，每步含干支、对日主十神、起止年份。
- `end_year = start_year + 10`（左闭右开，即下一步起始年），对齐《00》响应示例（1993→2003）。
- 时辰未知（`partial: true`）：起运距离按当地钟表 12:00 估算，误差 ≤ ±4 命理月内的天级；文档明示，模型B须在溯源层告知不确定性（文案归B）。

## 8. 时辰未知模式（partial）

- `hour = null`，`true_solar_time = null`，`partial: true`；五行按 6 字口径；日柱按公历出生日判定（无时辰即无子时归属争议）。

## 9. 流年与当前大运（current）

- 流年以立春为界：`current.annual.year` 为干支纪年之年份标签（如 2026-02-04 后为丙午）。
- `current.luck_cycle_index`：当前日期所处大运序号；**0 = 尚未起运**。

### 11.1 关键四十神 `key_ten_gods`（**恰 4 个**，选取规则）
按下列优先级取 4 个**不同**十神，各带 `position`（来源）与 `rank`（1–4）：
1. **rank1 = 月支本气十神**（`position=month_branch_main`）——月令为格局之首，最重。
2. 依次并入**透出天干十神**，顺序 **月干 > 时干 > 年干**（`position=<pillar>_stem`）；**日干=自身（比肩），不计**；跳过已选十神，补至 4 个。
3. 仍不足 4 个：按**地支藏干出现频次**（高→低）补入；同频者按**首次出现顺序**（柱序 年<月<日<时，支内 本气>中气>余气）优先（`position=<pillar>_branch_hidden`）。
- partial（无时辰）命例同规则；三柱含足够十神种类，实测仍恰满 4 个。

### 11.2 十神互动 `interactions`（对称，只出数据禁叙事）
对 `key_ten_gods` 两两组合（无序对，恰 C(4,2)=6 条）判关系，词表限 **生/克/化/竞**：
- **竞**：两者同属一功能组（同占一位，相互竞立），`note_cn=同气并立`。
- **化**：一为官杀、一为印 → 化克为生护身，`note_cn`=`杀印相生`（含七杀）或 `官印相生`。此特判先于"生"。
- **生**：二者功能组在相生环（印→比劫→食伤→财→官杀→印）相邻。`note_cn` ∈ {印生身, 比劫生食伤, 食伤生财, 财官相生}。
- **克**：二者在相克环（比劫→财→印→食伤→官杀→比劫）相邻。`note_cn` ∈ {比劫夺财, 财破印, 枭神夺食(偏印+食神)/印制食伤, 伤官见官(伤官+官)/食神制杀, 官杀攻身}。
- 关系与 a/b 顺序无关（对称）；`note_cn` 为经典简称（2–4字），非叙事文案。

### 11.3 流年三合 `annual_next3`
自当前流年 `current.annual.year` 起**连续 3 年**：`{year, stem, branch, ten_god}`。干支按 `(年−4) mod 10/12`，`ten_god` 以日干判年干。流年以立春为界（同 §9）。

### 11.4 夫妻宫 `relationship.spouse_palace`
**夫妻宫 = 日支**：`{branch=日支, hidden_ten_gods=日支藏干十神}`（供叙事层，**禁确定性预言**，宪法三）。partial 仍有日支，正常输出。

### 11.5 事业五组计数 `career`
`{authority(官杀), wealth(财), resource(印), output(食伤), peer(比劫)}`，**与五行 main 口径同基**：逐字（天干 + 地支**本气**）各计 1，按十神归入五组；**日干=自身→比肩→peer**。满盘和=8，partial（无时辰）和=6。十神→组映射见 `constants.TEN_GOD_CATEGORY`。
