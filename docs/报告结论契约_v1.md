# 报告结论契约 v1（ReportSegment）

> 状态：代码已落地为 `bazi_engine.report_contract`；传统规则与英文文案的命理顾问/安全编辑会签仍待补。  
> 适用：任何用户可见的报告正文、导出文件、First Scroll 样章、Full Codex、节气月提醒、付费页示例、AI 输出和人工精审交付。

## 目的

八字计算结果与“人生结论”不是同一类东西。`ReportSegment v1` 要求每一个面向用户的段落都带上：它是什么类型的陈述、依据从哪里来、采用哪个计算口径、以及它不应被理解成什么。

没有通过这个契约的文本不得发布；不能靠在页面末尾放一条总免责声明来替代单段依据和边界。

## 稳定数据结构

模型A 输出或校验后的单段 JSON 结构如下。字段名是模型B可依赖的稳定映射面；模型B不应重新猜测、改名或丢弃它们。

```json
{
  "contract_version": "report_segment_v1",
  "segment_id": "core-day-master",
  "claim_type": "traditional_reading",
  "body": "...",
  "boundary": "This is one symbolic traditional reading ...",
  "evidence": [
    {"kind": "chart_field", "ref": "day_master.element"}
  ],
  "trace_refs": ["meta.calculation_trace.profile_id"],
  "source_refs": ["editorial:day-master-guide-v1"],
  "profile_id": "zi_ping_solar_v1",
  "content_version": "2026-07-19",
  "risk_flags": ["identity"],
  "status": "published"
}
```

### 必填字段

| 字段 | 规则 |
|---|---|
| `segment_id` | 3–128 位稳定小写标识；一个报告内不得语义复用。 |
| `claim_type` | 必填，只能为下表值。 |
| `body` | 用户可见正文；会经过禁止结论扫描。 |
| `boundary` | 段落级边界文本；不可为空或用总免责声明替代。 |
| `evidence` / `trace_refs` / `source_refs` | 至少一个非空。传统解释须满足更严格的双重溯源。 |
| `profile_id` | 生成该阅读时的传统计算口径，例如 `zi_ping_solar_v1`。 |
| `content_version` | 模板/编辑内容版本，支持回溯与修订。 |
| `status` | `published` 或 `excluded`。后者只可配 `unsupported_or_excluded`。 |

`evidence.ref`、`trace_refs` 与 `source_refs` 必须是稳定、非敏感的引用标识；不得放入 `birth_date`、`birth_time`、`location`、经纬度、`true_solar_time` 或 `chart_id`。

## Claim type 与最低溯源要求

| claim_type | 适用内容 | 发布前最低要求 |
|---|---|---|
| `chart_fact` | 可机械复算的盘面事实 | `evidence` 或 `trace_refs`；不得写成结果预言。 |
| `traditional_reading` | 宪章正式术语：采用口径下的传统象征性阅读 | 至少一条 `evidence`，并同时有 `trace_refs` 或 `source_refs`。 |
| `traditional_interpretation` | 迁移兼容的传统解释名称；新模板优先使用 `traditional_reading` | 与 `traditional_reading` 同样严格。 |
| `reflection_prompt` | 用户可自主选择的反思问题 | 至少一个依据/Trace/来源；不得变成命令或“唯一解”。 |
| `educational_fact` | 术语、流派、典籍或历史的教育性说明 | 必须有 `source_refs`；不能把典籍当成科学验证。 |
| `unsupported_or_excluded` | 被拒绝的高风险请求的安全替代 | `status=excluded`，保留政策来源，不回显原始高风险文本。 |

`traditional_interpretation` 是为迁移时不让旧内容绕过校验而保留的兼容值；它不代表一套新增传统口径。命理顾问会签后的新模板应使用宪章名称 `traditional_reading`。

## 代码入口

```python
from bazi_engine.report_contract import make_report_segment, enforce_report_contract

segment = make_report_segment(
    segment_id="core-day-master",
    claim_type="traditional_reading",
    body="Under this profile, this pattern is traditionally read as a theme for reflection.",
    evidence=[{"kind": "chart_field", "ref": "day_master.element"}],
    trace_refs=["meta.calculation_trace.profile_id"],
    source_refs=["editorial:day-master-guide-v1"],
    profile_id="zi_ping_solar_v1",
    content_version="2026-07-19"
)
```

默认模式是严格失败：结构缺失、缺依据或正文命中禁区都会抛出 `ReportContractError`/`ReportPolicyViolation`。仅在服务端明确选择 `on_policy_violation="degrade"` 时，命中政策的单段可替换为 `unsupported_or_excluded`；结构错误永远不得被降级掩盖。

## 已编码的禁止输出门

用户可见正文会拒绝或降级以下类别：

- 宿命绝对化、保证和不可避免的结果；
- 把八字或命理包装为科学已证实；
- 疾病、心理健康、怀孕/生育的判断、预测、诊断、治疗；
- 投资、借贷、买卖、房产、博彩与收益指令；
- 法律结论或诉讼/签约指令；
- 自伤、自杀、伤害他人与报复；
- 基于性别、性取向、跨性别身份、残障或种族的贬低、角色判决或能力排除。

这是发布门的基线，不是关键词游戏。新型同义说法、恐惧营销、典故暗示或跨语言变体一经发现，安全编辑可直接下线，并应补入阴性测试。

## 模型B 必须完成的接入（尚未由模型A代做）

模型A当前只输出排盘和 FullChartData，并不生成用户叙事；因此模型B（及未来 LLM/人工交付链）必须把这套契约接到**所有**用户可见出口：

1. 模板、LLM 结果、按钮文案、付费页样章、导出和通知统一映射成 `ReportSegment`；禁止直接渲染裸字符串。
2. 在服务端调用 `enforce_report_contract(..., on_policy_violation="raise")`。对实时问答若采用降级，也只能使用 `degrade`，并记录类别，不能回显被拦截正文。
3. UI 必须同时显示 `claim_type` 的人类可读标签、`boundary` 和可展开的 `evidence/trace/source`；不能只在开发日志保存。
4. `trace_refs` 必须能映射到本次报告的 ChartResult / FullChartData，而不是静态占位符；`source_refs` 必须落到版本化术语库或已核对的典籍/编辑来源。
5. 支付墙内外、分享图和邮件推送同样必须走校验；不得因渠道不同删除边界或替换成“更准/改命/保证”话术。
6. 每次内容版本升级都必须保留 `profile_id + content_version + segment_id`，以便用户和客服复核。

## 不由本契约声称完成的事

- 结构合法不等于传统解释已获命理顾问认可；新增规则、典籍引用和英文转写仍须会签。
- 自动拦截不替代安全编辑审阅，尤其不能替代对暗示、上下文、文化挪用和恐惧营销的判断。
- 本模块不处理账户权限、付款、退款、数据删除、人工服务 SOP 或生产监控；这些由盈利与可信上线方案中的其他 P0 工作项负责。
